# v0.3 design & handoff: live streaming dictation

> **Status (2026-06-14): v0.3 shipped as growing-window streaming + VAD (append-only).**
> What actually shipped, and why:
> - **Growing-window re-transcription.** Each pass re-transcribes the *whole utterance so
>   far* (`_stream_loop`), so whisper always has full context = batch-grade quality, and
>   types only the words that have *stabilised* across consecutive passes (`_common_prefix`,
>   `_emit_words`). Append-only, no backspacing. `max_context_sec` re-anchors to bound cost;
>   `_final_flush` does one last full-context pass at STOP.
> - **Why not chunks.** The first cut shipped fixed ~2.5s *chunks* (transcribe only the new
>   slice). It worked but **tanked quality**: each chunk had no context, and whisper's
>   accuracy is mostly context. User feedback ("one step forward two steps back") was right;
>   the growing window restores quality while staying live.
> - **VAD: removed, then re-added with onset-friendly tuning.** Server-side `--vad` (silero)
>   kills silence hallucinations, but the *default* tuning trims quiet speech onsets, eating
>   the first word after every pause on this low-output mic (measured floor 0.025, speech
>   0.026-0.045 RMS; default VAD dropped "ask not" on attenuated jfk). It was briefly removed
>   in 0.3.0, relying on the stabilise-across-passes rule, but that is NOT enough: on
>   sustained silence whisper repeats the *same* hallucination ("Thank you for watching",
>   and random languages like Japanese under `language=auto`), which is "stable", so it
>   commits, defeats the auto-stop and runs the session away. 0.3.1 brings VAD back tuned for
>   onsets: `--vad-threshold 0.25 --vad-speech-pad-ms 500 --vad-min-speech-duration-ms 0`.
>   Verified: pure noise -> empty, attenuated jfk -> onsets intact ("ask not ... Ask what").
> - **language = "auto".** Pinning `pl` broke English and mixed PL/EN. Auto is most reliable
>   with full context (which the growing window restores), so it's the default again. A
>   language picker in the indicator menu is the planned v0.4 follow-up (user request).
> - **Capture-live gate.** START holds the red indicator until pw-record is actually
>   recording, so a fast speaker doesn't lose the first word to device warm-up (~50ms).
>
> - **Non-ASCII typing.** ydotool 1.x injects raw US-layout keycodes and silently drops
>   Polish diacritics. xdotool *can* type Unicode (XTEST keysym remap) but its remap storm
>   **froze the X server** here, so it is not used. Instead `_type_text` types ASCII via
>   ydotool and inserts any non-ASCII piece byte-exact via clipboard + Ctrl+Shift+V.
>
> Shipped code: `daemon.py` (`_stream_loop`, `_emit_words`, `_final_flush`, `_common_prefix`,
> `_transcribe_pcm`, `_type_text` ASCII/clipboard split), `config.py` (`mode`,
> `stream_interval_sec`, `max_context_sec`), `systemd/whisper-server.service` (onset-tuned
> `--vad*`), `install/02`+`03` (VAD model), tests in `tests/test_streaming.py`. Deferred to
> **v0.4**: backspace-correcting the tentative tail, smarter window anchoring, language
> picker, smart line breaks, a Wayland-friendly Unicode typer (`wtype`). The original design
> notes below are kept as the v0.4 starting point.

This is a working design + handoff note for the v0.3 effort. v0.2.x is shipped and public
(`main`, tag `v0.2.1`). Read this together with `ROADMAP.md` and `CHANGELOG.md`.

## Goal

Make text appear and self-correct **as you speak**, like macOS dictation, instead of
arriving all at once when you stop. Today dictation is batch: record the whole utterance,
then transcribe once at STOP. v0.3 turns this into a streaming loop.

## Current architecture (what you're changing)

All code is in this repo now (the real app, not the old scaffolding).

- `src/dicti/daemon.py` , the daemon. State machine `IDLE -> LISTENING -> PROCESSING -> IDLE`.
  Key methods:
  - `start_recording()` , spawns `pw-record --rate=16000 --format=s16 --channels=1` writing
    to `TMP_WAV` (`/tmp/dictation-record.wav`); starts the safety timer and `_monitor_silence`.
  - `_monitor_silence()` / `_tail_rms()` , reads raw bytes from the **tail** of the growing
    WAV (the WAV header frame-count is unreliable mid-record) and auto-stops after
    `silence_timeout_sec` of sub-threshold RMS.
  - `stop_and_transcribe()` , aborts the recorder, then `_transcribe()` -> `_clean_transcript()`
    -> `_insert()`.
  - `_has_speech()` , energy gate: skips transcription if no window exceeds the speech RMS
    threshold (kills silence hallucinations).
  - `_clean_transcript()` , drops non-speech tokens (`[Silence]`, etc.), hallucination
    phrases (`_HALLUCINATION_PHRASES`), and collapses per-segment newlines.
  - `_transcribe()` , POSTs the WAV to whisper-server `/inference` over HTTP, returns text.
  - `_insert()` / `_type_text()` , types via `ydotool type --key-delay N --file -` (stdin,
    UTF-8, escapes off); also copies to clipboard via `xclip` as a safety net.
  - `serve()` , unix socket on `$XDG_RUNTIME_DIR/dictation.sock`, commands
    START/STOP/TOGGLE/CANCEL/STATUS.
  - `_set_state()` writes the current state to `$XDG_RUNTIME_DIR/dictation.state`.
- `src/dicti/config.py` , `Config` dataclass loaded from `~/.config/dicti/config.toml`
  (stdlib `tomllib`), defaults baked in.
- `gnome-extension/dicti@local/` , the top-bar indicator (GNOME Shell extension). Reads the
  state file, swaps icon, spinner during PROCESSING, click toggles via `dictate-toggle`.
  Lives in the **center** box so GNOME's privacy mic indicator (right box) doesn't shove it.
- `src/dicti/indicator.py` + `systemd/dicti-indicator.service` , OPTIONAL AppIndicator
  fallback for non-GNOME; not enabled by default.
- `systemd/whisper-server.service` , whisper.cpp HTTP server, Vulkan, `ggml-medium-q5_0`,
  `127.0.0.1:8178`, `--suppress-nst`. Request/response only (no streaming endpoint).
- `install/00..07` + `install/install.sh` , guided install. Units are templated with `@SRC@`
  (substituted to the repo `src/` at install) and run `python3 -m dicti.daemon`.

Flow today: `key -> keyd -> GNOME shortcut -> dictate-toggle -> socket -> daemon ->
pw-record -> WAV -> HTTP -> whisper-server (Vulkan) -> clean -> ydotool type`.

## Proposed v0.3 design

1. **Stream audio.** Run `pw-record` to **stdout** (`... -` / target `-`) and read raw s16le
   PCM in a thread into a rolling buffer, instead of only writing a file. Keep the file too
   if useful for a final full-context pass.
2. **Windowed transcription loop.** Every ~1.5,2.5s, build a small WAV in memory from a
   sliding/growing window of the buffer and POST it to whisper-server. (The server has no
   streaming endpoint; the daemon orchestrates windows.) Longer context = more accurate, so
   re-transcribing a growing window lets later passes fix earlier words.
3. **Committed vs tentative text.** Track what has already been typed. Each pass:
   - Diff the new transcription against the already-typed text.
   - Type the new suffix (delta).
   - If earlier **tentative** words changed, backspace-correct: `ydotool key 14:1 14:0`
     (KEY_BACKSPACE=14) x N, then type the correction. Only correct words not yet "committed"
     (e.g. older than the last VAD boundary), so we never rewrite far back.
4. **VAD (strongly recommended).** Run whisper-server with `--vad` + a silero VAD model
   (download via whisper.cpp's model script). VAD only feeds voiced audio to the model: it
   fixes silence hallucinations at the source and gives natural utterance boundaries to
   "commit" text on. This is the robust replacement for v0.2.1's energy gate + blocklist.

### Decision the next agent must settle with the user first

- **Full in-place correction (backspacing) vs append-only-stable.** Backspacing is the most
  Mac-like but risky: if the user moves the cursor or focus, backspaces eat the wrong text.
  A safer first increment is **append-only**: only type a segment once VAD/diff says it's
  stable, appending finalized text live without ever backspacing. Recommend shipping
  append-only first, then layer in bounded correction.
- **Latency target** and whether to keep **batch mode** as a config fallback (`mode = "batch"
  | "streaming"`).
- VAD model download/build acceptable? (extra step in `install/02`/`03`.)

## Gotchas discovered (save yourself time)

- whisper-server `--help` (on this build) has `--suppress-nst`, `--vad`, `--no-speech-thold`,
  `--prompt`. `language=auto` works and PL/EN auto-detect is reliable for clear speech;
  low-confidence detection on soft audio is what produced `[Foreign Language]`.
- Whisper text-hallucinates "Thanks for watching", "The end", subtitle credits on silence;
  these are NOT bracketed tokens, so they need the phrase blocklist / VAD, not `--suppress-nst`.
- pw-record's WAV header is not finalized mid-record; for the buffer, parse raw PCM from
  stdout (s16le, 16 kHz, mono) rather than trusting the header.
- `ydotool` typing/backspacing only affects the **focused** window. Continuous live typing
  makes "user switched apps" worse than batch. Consider detecting/locking the target window,
  or at least pausing insertion when focus changes (X11: `xdotool getactivewindow`).
- Insertion of newlines = Enter, which auto-indents in code editors (the v0.2.1 "staircase").
  v0.3 should decide real line breaks deliberately (see ROADMAP).
- State machine needs rework: streaming overlaps LISTENING and typing. Add states or a
  sub-state and extend the state file + extension `ICONS`/`_update` accordingly. The
  extension already polls the state file every 2s and on inotify change.
- Spinner is `Animation.Spinner` (GNOME). The extension UUID is `dicti@local`.

## Dev loop / deploy on this machine

- Env: Debian, X11, GNOME Shell 48.7, integrated Vulkan GPU. `ydotoold` runs,
  `YDOTOOL_SOCKET=/run/user/1000/.ydotool_socket`. whisper-server + dictation are user services.
- Daemon: edit `src/`, then `systemctl --user restart dictation` (runs `python3 -m dicti.daemon`
  with `PYTHONPATH=<repo>/src`). Logs: `journalctl --user -u dictation -u whisper-server -f`.
- whisper-server unit change: `install -m644 systemd/whisper-server.service
  ~/.config/systemd/user/ && systemctl --user daemon-reload && systemctl --user restart
  whisper-server`. Confirm Vulkan in the journal; the daemon also warns on CPU fallback.
- Extension change: `cp -r gnome-extension/dicti@local/. ~/.local/share/gnome-shell/extensions/dicti@local/`
  then reload GNOME Shell (human: Alt+F2, `r`). Can't reload the shell non-interactively.
- Tests: prefer pure-stdlib unit tests. Pattern used so far: stub the `requests` module,
  set `XDG_RUNTIME_DIR`/`XDG_CONFIG_HOME` to temp dirs, build synthetic WAVs with `wave`,
  call methods directly (see how `_clean_transcript`/`_has_speech`/`_tail_rms` were tested).
  Consider adding a real `tests/` dir for v0.3.
- Git: branch off `main`, fast-forward merge, push over HTTPS (`origin` is HTTPS). Commit
  messages clean, no AI attribution (enforced in the user's global config).

## Definition of done (v0.3)

- Speaking produces text that appears within ~1,3s and refines as you continue.
- No silence hallucinations (VAD).
- Long sessions still bounded by `max_record_sec`; silence auto-stop still works.
- Configurable mode (at least streaming on/off), documented in `config.toml.example`.
- README/ROADMAP/CHANGELOG updated; verified live on this machine end to end.
